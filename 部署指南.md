# 🛠️ 办公工具箱 — 小程序 / 桌面应用 / 永久网址 全攻略

你的项目是一个 Flask 网页版办公工具箱，要变成小程序、桌面应用或永久网址，各有不同的路径。下面逐一讲清楚。

---

## 方案一：永久网址（最简单，推荐先做）

### 原理
把 Flask 应用部署到云平台，别人通过 URL 就能访问。

### 关键改动
你原来的版本需要用户输入**本地文件夹路径**（如 `D:\发票`），这在线上无法使用。我已帮你改了一版 `app_cloud.py`，核心改动：
- 文件操作 → 改为**用户上传文件**（拖拽/选择）
- 处理结果 → 提供**下载链接**
- HTML 模板 → 内嵌在 Python 里，不需要 `templates/` 目录
- 数据库 → SQLite，云端也可用

### Render 部署步骤（免费）

1. **把代码推到 GitHub**
   ```bash
   cd toolbox_cloud
   git init
   git add .
   git commit -m "办公工具箱云部署版"
   git remote add origin https://github.com/你的用户名/toolbox.git
   git push -u origin main
   ```

2. **在 Render 上创建服务**
   - 打开 https://render.com ，用 GitHub 登录
   - 点 "New" → "Web Service"
   - 选择你的 GitHub 仓库
   - 配置：
     - **Build Command**: `pip install -r requirements.txt`
     - **Start Command**: `gunicorn app_cloud:app`
     - **Environment**: 添加 `PYTHON_VERSION` = `3.11.0`
   - 点 "Create Web Service"

3. **等 2-3 分钟自动部署完成**，你会得到一个永久 URL：
   ```
   https://toolbox-xxxx.onrender.com
   ```

### Railway 部署步骤（更稳定）

1. 打开 https://railway.app ，GitHub 登录
2. "New Project" → "Deploy from GitHub repo"
3. 选择仓库，自动识别 `Procfile`
4. 部署完成后点 "Settings" → "Generate Domain"
5. 得到永久 URL：`https://toolbox-production-xxxx.up.railway.app`

> **注意**：免费版 Render 服务 15 分钟无访问会休眠，首次访问需等 ~30 秒唤醒。Railway 更稳定但免费额度有限。

## 方案二：桌面应用（PyInstaller 打包）

### 原理
用 PyInstaller 把 Python 代码打包成 `.exe`（Windows）/ `.app`（macOS），双击就能运行。

### 适合你的原始 `toolbox.py`（命令行版）或 `app.py`（网页版）

#### 打包步骤

1. **安装 PyInstaller**
   ```bash
   pip install pyinstaller
   ```

2. **打包命令行版**（原始 `toolbox.py`）
   ```bash
   pyinstaller --onefile --name="办公工具箱" toolbox.py
   ```
   生成 `dist/办公工具箱.exe`，双击即可运行。

3. **打包网页版**（`app.py`，带界面更美观）
   ```bash
   pyinstaller --onefile --add-data "templates;templates" --add-data "toolbox.db;." --name="办公工具箱" app.py
   ```

4. **如果想隐藏命令行黑窗口**
   ```bash
   pyinstaller --onefile --noconsole --add-data "templates;templates" --name="办公工具箱" app.py
   ```

#### 分发方式
- 直接把 `dist/办公工具箱.exe` 发给别人，无需安装 Python
- 用户双击运行后，自动打开浏览器 `http://127.0.0.1:5000`

#### 更进一步：用 Electron 包装
如果你想要更专业的桌面应用体验（带窗口框架、系统托盘、自动打开浏览器）：

```javascript
// main.js (Electron 入口)
const { app, BrowserWindow } = require('electron');
const { spawn } = require('child_process');
const path = require('path');

let win;
let server;

app.on('ready', () => {
    // 启动 Flask 后端
    server = spawn(path.join(__dirname, '办公工具箱.exe'));
    
    // 等 2 秒让服务启动
    setTimeout(() => {
        win = new BrowserWindow({
            width: 1200, height: 800,
            title: '🛠️ 办公自动化工具箱',
            icon: path.join(__dirname, 'icon.png'),
        });
        win.loadURL('http://127.0.0.1:5000');
        win.on('closed', () => { server.kill(); app.quit(); });
    }, 2000);
});
```

## 方案三：微信小程序（工程量最大）

### 原理
微信小程序不能直接运行 Python，需要：
1. **后端**：你的 Flask 应用部署到云（方案一的 URL）
2. **前端**：用微信小程序框架重写界面，调用后端 API

### 架构

```
微信小程序（前端）          云端 Flask（后端）
┌──────────────┐           ┌──────────────┐
│  上传文件页面  │──上传──→  │ /api/merge   │
│  合并Excel页面 │←─结果──  │ /api/invoice │
│  发票提取页面  │          │ /api/email   │
│  ...          │          │ ...          │
└──────────────┘           └──────────────┘
```

### 小程序前端核心代码示例

```javascript
// pages/merge/merge.js
Page({
  data: { files: [], result: '' },
  
  // 选择文件
  chooseFile() {
    wx.chooseMessageFile({
      count: 10,
      type: 'file',
      extension: ['xlsx', 'xls'],
      success: res => {
        this.setData({ files: res.tempFiles });
      }
    });
  },
  
  // 合并请求
  doMerge() {
    const { files } = this.data;
    if (!files.length) return wx.showToast({ title: '请先选择文件', icon: 'none' });
    
    wx.showLoading({ title: '合并中...' });
    
    // 逐个上传文件
    const uploadTasks = files.map(f => new Promise((resolve, reject) => {
      wx.uploadFile({
        url: 'https://你的域名/api/merge-excel',
        filePath: f.path,
        name: 'files',
        formData: { mode: '1' },
        success: res => resolve(JSON.parse(res.data)),
        fail: reject
      });
    }));
    
    Promise.all(uploadTasks).then(results => {
      wx.hideLoading();
      const data = results[results.length - 1];
      if (data.success) {
        // 下载结果
        wx.downloadFile({
          url: `https://你的域名/download/${data.download_id}`,
          success: res => {
            wx.openDocument({ filePath: res.tempFilePath });
          }
        });
      }
    });
  }
});
```

### 小程序注意事项

| 事项 | 说明 |
|------|------|
| **域名备案** | 小程序要求后端域名必须 HTTPS + 已备案 |
| **文件大小** | 小程序上传文件限制 100MB |
| **开发成本** | 需要学习微信小程序框架（类似 Vue），前端要全部重写 |
| **审核** | 提交微信审核，通常 1-3 天 |

### 快捷方案：用 Coze 发布

如果你不想写小程序代码，**最省事的方式**：

1. 在 Coze 上创建一个 Bot
2. 把工具箱功能做成 Coze 的 Plugin / Workflow
3. 发布到微信/飞书等渠道
4. 用户直接在聊天界面使用

## 方案对比总结

| 方案 | 难度 | 时间 | 适合场景 | 文件操作 |
|------|------|------|---------|---------|
| **永久网址** | ⭐ | 30分钟 | 分享给别人用 | 需改为上传模式 ✅已改好 |
| **桌面应用** | ⭐⭐ | 1小时 | 自用/离线使用 | 完整保留 ✅ |
| **微信小程序** | ⭐⭐⭐⭐ | 3-5天 | 微信生态分发 | 需前后端分离 |
| **Coze 发布** | ⭐⭐ | 1-2小时 | 聊天式使用 | 部分功能受限 |

## 推荐路径

**先做永久网址** → 我已经帮你把 `app_cloud.py` 改好了，HTML 全部内嵌，文件操作改为上传模式，推到 GitHub + Render 部署 30 分钟搞定。

**再考虑桌面应用** → 用 PyInstaller 打包原始 `app.py`，一个命令出 `.exe`。

**小程序是锦上添花** → 如果确实有微信生态需求再投入。
